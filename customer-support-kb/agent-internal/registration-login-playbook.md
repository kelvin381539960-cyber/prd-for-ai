# 注册与登录 Playbook

## 适用场景

- 用户无法注册
- 用户无法登录
- 用户收不到验证码
- 用户忘记密码
- 用户账户疑似异常

## 客服先确认

1. 用户使用的是邮箱还是手机号。
2. 用户当前停留在哪个页面。
3. 页面是否有错误提示。
4. 用户是否多次请求验证码。
5. 用户是否能访问原邮箱或手机号。

## 可向用户询问的信息

- 注册邮箱或手机号
- 问题发生时间
- 页面截图
- 错误提示文案
- 是否更换过设备或网络

## 建议处理步骤

1. 引导用户检查填写信息和网络连接。
2. 引导用户检查垃圾邮件、短信拦截或邮箱拦截。
3. 如多次尝试失败，升级给人工客服进一步核查。
4. 如涉及账户安全或疑似盗号，必须升级。

## 可使用的用户回复口径

```text
请先确认邮箱或手机号是否填写正确，并检查网络连接、短信拦截或邮箱垃圾邮件。如果多次尝试后仍无法收到验证码，建议联系人工客服协助检查。
```

## 必须升级的情况

- 用户无法访问原邮箱或手机号。
- 用户怀疑账户被盗。
- 页面提示账户受限或冻结。
- 用户无法完成密码重置。

## 不要向用户透露

- 内部账户状态字段。
- 安全或风控判断逻辑。
- 验证码系统内部规则。
