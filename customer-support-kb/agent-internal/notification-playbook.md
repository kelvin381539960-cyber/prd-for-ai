# 通知与系统邮件 Playbook

## 适用场景

- 用户收不到 App 推送
- 用户收不到系统邮件
- 用户收不到验证码邮件
- 用户怀疑邮件或通知异常

## 客服先确认

1. 问题类型：App 通知、系统邮件、验证码邮件或安全通知。
2. 用户邮箱或手机号是否正确。
3. 用户是否检查垃圾邮件、短信拦截或系统通知权限。
4. 用户是否多次尝试重新发送。
5. 是否存在疑似钓鱼邮件或异常通知。

## 可向用户询问的信息

- 注册邮箱或手机号
- 问题发生时间
- 邮件标题或通知截图，注意不要要求敏感信息
- 是否检查垃圾邮件或通知权限
- 是否更换设备或邮箱设置

## 建议处理步骤

1. 引导用户检查通知权限、网络和邮箱垃圾箱。
2. 引导用户确认邮箱或手机号是否正确。
3. 多次尝试仍失败时，升级人工。
4. 疑似钓鱼或异常通知，必须升级。

## 可使用的用户回复口径

```text
请先确认邮箱或手机号是否正确，并检查垃圾邮件、短信拦截、系统通知权限和网络连接。如果多次尝试后仍收不到通知或邮件，建议联系人工客服协助检查。
```

## 必须升级的情况

- 用户持续收不到验证码、系统邮件或安全通知。
- 用户怀疑收到钓鱼邮件。
- 用户需要修改邮箱或手机号。
- 通知内容与账户操作不一致。

## 不要向用户透露

- 内部邮件发送规则。
- 未确认的官方邮件域名。
- 内部通知模板或触发条件。
