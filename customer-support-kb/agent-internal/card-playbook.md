# 卡 Playbook

## 适用场景

- 卡申请失败
- 卡申请审核中
- 卡被冻结或无法使用
- 卡交易失败
- 重复扣款或金额异常
- 用户询问费用、限额或支持地区

## 客服先确认

1. 用户问题类型：申请、管理、交易、冻结、费用/限额咨询。
2. 用户是否已完成身份验证。
3. App 页面显示的卡状态。
4. 如为交易问题，确认时间、金额、商户名称。
5. 是否有页面截图或错误提示。

## 可向用户询问的信息

- 卡申请或交易发生时间
- 页面截图
- 错误提示
- 交易金额
- 商户名称
- App 内卡状态

## 建议处理步骤

1. 引导用户查看 App 页面提示和卡状态。
2. 如页面提示补充资料，按页面提示操作。
3. 如涉及交易失败、重复扣款、金额异常，升级人工。
4. 费用、限额、支持地区未确认时，不直接回答。

## 可使用的用户回复口径

```text
请先查看 App 页面是否有具体提示。如果是卡交易失败、重复扣款或金额异常，建议联系人工客服，并准备好交易时间、金额、商户名称和页面截图。
```

## 必须升级的情况

- 卡申请失败或长时间无结果。
- 卡被冻结或无法使用。
- 交易失败、重复扣款、金额异常。
- 用户要求取消、关闭、恢复或补发卡。
- 未确认的费用、限额、支持地区问题。

## 不要向用户透露

- 内部卡供应商规则。
- 风控或审核逻辑。
- 未确认的费用、限额和地区支持。
- 扣款责任结论。
