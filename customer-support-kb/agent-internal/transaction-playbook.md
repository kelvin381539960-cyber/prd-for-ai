# 交易记录 Playbook

## 适用场景

- 用户找不到交易记录
- 交易状态长时间处理中
- 交易失败
- 交易金额、时间或状态异常

## 客服先确认

1. 交易类型：充值、发送、兑换、卡交易或其他。
2. 交易时间。
3. 交易金额。
4. App 当前显示状态。
5. 是否有 TxID 或页面截图。

## 可向用户询问的信息

- 交易类型
- 交易时间
- 金额和币种
- TxID / 交易哈希，如适用
- 页面截图
- 当前交易状态

## 建议处理步骤

1. 引导用户确认账户、资产、时间范围和交易类型是否正确。
2. 引导用户刷新页面。
3. 如交易长时间处理中或状态异常，升级人工。
4. 涉及撤销、追回、赔偿时，不承诺处理结果，升级人工。

## 可使用的用户回复口径

```text
请先确认你查看的是正确的账户、资产、时间范围和交易类型。如果交易长时间显示处理中，或金额、状态与预期不一致，建议联系人工客服协助核查。
```

## 必须升级的情况

- 交易长时间处理中。
- 用户找不到交易记录。
- 交易金额、时间或状态异常。
- 用户要求撤销、追回或赔偿。

## 不要向用户透露

- 内部交易状态机。
- 内部账务处理逻辑。
- 未确认的状态定义或处理时长。
