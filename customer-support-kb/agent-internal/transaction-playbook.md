# 交易记录 Playbook

## 适用场景

- 用户找不到交易记录。
- 交易状态长时间处理中。
- 交易失败。
- 交易金额、时间、币种、商户或状态异常。
- 卡退款未到账或金额异常。
- 交易记录页面提示数据异常、网络异常或加载失败。

## 客服先确认

1. 交易类型：钱包充值、接收、发送、兑换、卡消费、卡退款、申卡扣费或退款。
2. 用户查看的是全量交易、卡交易、钱包交易还是交易详情。
3. 用户是否选择了正确的账户、卡、资产、时间范围或交易类型。
4. 交易时间、金额、币种、商户名称。
5. App 当前显示状态。
6. 是否有 TxID、交易 ID 或页面截图。
7. 页面是否提示网络异常、数据异常或服务异常。

## 可向用户询问的信息

- 交易类型。
- 交易时间。
- 金额和币种。
- 卡交易商户名称。
- TxID / 交易哈希，如适用。
- App 交易 ID 或页面截图。
- 当前交易状态。
- 用户是否刷新页面或切换筛选条件。

## 建议处理步骤

1. 引导用户确认账户、卡、资产、时间范围和交易类型是否正确。
2. 引导用户刷新页面或稍后重试。
3. 钱包充值问题：要求提供 TxID、币种、网络、金额和时间。
4. 卡交易问题：要求提供交易时间、金额、商户名称和截图。
5. 退款问题：不承诺到账时间、入账位置或手续费退还规则，先收集信息并升级。
6. 页面数据异常或网络异常：先做通用排查，持续异常则升级。
7. 涉及撤销、追回、赔偿或责任判断时，不承诺结果，升级人工。

## 可使用的用户回复口径

```text
请先确认你查看的是正确的账户、卡、资产、时间范围和交易类型。如果交易长时间显示处理中，或金额、状态与预期不一致，建议联系人工客服协助核查。
```

```text
如果页面提示网络异常或数据异常，请先检查网络并刷新页面。如果问题持续存在，建议联系人工客服协助检查。
```

## 必须升级的情况

- 交易长时间处理中。
- 用户找不到交易记录。
- 交易金额、时间、状态、币种或商户异常。
- 卡退款长时间未到账或金额不一致。
- 充值链上成功但 App 未到账。
- 用户要求撤销、追回、赔偿或责任判断。
- 页面持续数据异常或无法加载。

## 不要向用户透露

- 内部交易状态机。
- 内部账务、清结算或通道处理逻辑。
- 未确认的状态定义、查询范围、到账时间或退款规则。
- 对退款、扣款、失败原因的责任结论。
