# KYC 与钱包开户 Playbook

## 适用场景

- 用户需要完成 KYC
- KYC 审核中
- KYC 失败
- 用户无法重新提交资料
- 用户询问钱包开户条件

## 客服先确认

1. 用户当前 KYC 状态。
2. 页面是否提示需要补充资料。
3. 用户是否已提交证件或自拍信息。
4. 页面是否展示失败原因。
5. 用户是否需要修改身份信息。

## 可向用户询问的信息

- App 页面截图
- 页面提示文案
- 提交时间
- 是否能看到重新提交入口
- 是否使用了真实、清晰、完整的信息

## 建议处理步骤

1. 如果用户未提交，引导用户按 App 页面提示提交。
2. 如果审核中，引导用户等待页面更新，避免承诺具体时长。
3. 如果失败，引导用户查看页面提示并按提示重新提交。
4. 如果无法重新提交或涉及身份信息修改，升级人工。
5. 涉及国家/地区、合规、制裁或风险判断时，不直接回答，升级人工。

## 可使用的用户回复口径

```text
请按照 App 页面提示提交真实、清晰、完整的身份信息。如果页面显示审核中，请耐心等待；如果页面提示需要补充资料，请按提示重新提交。若无法重新提交或不清楚失败原因，建议联系人工客服协助处理。
```

## 必须升级的情况

- KYC 长时间无结果。
- KYC 失败且用户无法判断原因。
- 用户要求修改身份信息。
- 用户无法重新提交。
- 涉及合规或地区限制。

## 不要向用户透露

- 具体审核规则。
- 风控或合规判断逻辑。
- 如何提高或保证通过率的方法。
- 内部供应商或审核流程细节。
