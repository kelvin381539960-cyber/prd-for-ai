# KYC 与钱包开户 Playbook

## 适用场景

- 用户需要完成 KYC。
- KYC 审核中。
- KYC 失败。
- 用户无法重新提交资料。
- 用户无法继续钱包开户流程。
- 用户选择居住国家后进入 Waitlist 或被拦截。
- 证件、人脸或地址证明验证失败。

## 客服先确认

1. 用户当前 KYC 阶段：未开始、选择国家、证件验证、人脸验证、地址证明、审核中、失败、已通过、Waitlist。
2. 页面是否提示需要补充资料。
3. 用户是否能看到重新提交入口。
4. 用户失败的是证件、人脸还是地址证明。
5. 页面是否出现网络异常、系统异常或加载失败。
6. 用户是否需要修改身份信息。
7. 用户是否询问国家/地区支持、合规限制或审核规则。

## 可向用户询问的信息

- App 页面截图。
- 页面提示文案。
- 提交时间。
- 当前验证阶段。
- 是否能看到重新提交入口。
- 上传文件格式和大小是否符合页面提示。
- 是否使用真实、清晰、完整的信息。

## 建议处理步骤

1. 如果用户未提交，引导用户按 App 页面提示开始 KYC。
2. 如果用户被 Waitlist 拦截，引导用户以页面提示为准，不承诺支持范围。
3. 如果证件验证失败，引导用户重新拍摄/上传清晰、完整、无遮挡、无反光的有效证件。
4. 如果人脸验证失败，引导用户检查光线、面部无遮挡、保持稳定，并按照页面指引重试。
5. 如果地址证明失败，引导用户检查文件格式、大小、清晰度、姓名、地址和签发时间等页面要求。
6. 如果审核中，不承诺具体完成时间。
7. 如果无法重新提交、失败原因不清楚、长时间无结果，升级人工。
8. 涉及国家/地区、合规、制裁或风险判断时，不直接回答，升级人工。

## 可使用的用户回复口径

```text
请按照 App 页面提示提交真实、清晰、完整的信息。如果页面提示需要补充资料，请根据提示重新提交。若无法重新提交、审核长时间无结果，或不清楚失败原因，建议联系人工客服协助处理。
```

```text
如果页面提示当前国家或地区暂不支持，请以 App 页面展示为准。你可以按照页面提示加入 Waitlist，或联系人工客服确认当前状态。
```

## 必须升级的情况

- KYC 长时间无结果。
- KYC 失败且用户无法判断原因。
- 用户要求修改身份信息。
- 用户无法重新提交。
- 证件、人脸或地址证明多次失败。
- 涉及合规或地区限制。
- 页面提示账户或安全限制。

## 不要向用户透露

- 具体审核规则。
- 风控、合规或供应商判断逻辑。
- 如何提高或保证通过率的方法。
- 内部状态机、错误码或接口字段。
- 供应商处理细节。
