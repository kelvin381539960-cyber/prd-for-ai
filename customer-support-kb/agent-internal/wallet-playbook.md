# 钱包 Playbook

## 适用场景

- 用户资产余额异常。
- 用户找不到某个资产。
- 充值不到账。
- 充值时选错网络、地址或币种。
- Send / 发送失败或处理中。
- Swap / 兑换失败或处理中。
- 用户询问支持资产、网络、费用、限额、汇率或到账时间。

## 客服先确认

1. 用户问题类型：资产显示、充值、发送、兑换、交易记录。
2. 币种、网络、金额、地址是否正确。
3. 是否有 TxID / 交易哈希。
4. 链上交易是否已提交或确认。
5. App 内交易状态是什么。
6. 用户是否满足页面显示的最小充值或余额要求。
7. 用户是否使用第三方钱包或交易所。
8. 用户是否选择了页面提示的正确网络。

## 可向用户询问的信息

- 充值或交易币种。
- 网络 / 链。
- 金额。
- 交易时间。
- TxID / 交易哈希。
- App 页面截图。
- App 内交易状态。
- 第三方钱包或交易所的交易状态截图。

## 建议处理步骤

1. 资产问题：引导用户刷新页面，确认是否近期发生充值、发送、兑换或卡交易。
2. 充值问题：确认币种、网络、地址、金额、最低充值要求和链上状态。
3. 发送问题：确认收款人、金额、资产、余额和验证结果。
4. 兑换问题：确认卖出资产、买入资产、金额、汇率页面和结果状态。
5. 如果链上已确认但 App 未到账，升级人工。
6. 如果疑似错链、错地址、错币种，不承诺找回，升级人工。
7. 未确认的支持资产、支持网络、费用、限额、汇率和到账时间，不直接回答。

## 可使用的用户回复口径

```text
请先确认币种、网络、地址和金额是否正确，并查看交易状态。如果链上已确认但 App 内长时间未到账，建议联系人工客服，并准备好交易哈希、币种、网络、金额和时间。
```

```text
如果你可能选择了错误的网络、地址或币种，请尽快联系人工客服。区块链转账通常无法保证撤回或找回，具体情况需要进一步核查。
```

## 必须升级的情况

- 充值长时间不到账。
- 链上成功但 App 未到账。
- 错链、错地址、错币种。
- 资产余额异常。
- 发送或兑换失败后资产状态异常。
- 交易长时间处理中。
- 用户要求追回、撤销或赔偿。

## 不要向用户透露

- 内部账务、交易状态机或风控逻辑。
- 未确认的支持币种、网络、手续费、限额、汇率。
- 找回、撤销或赔付承诺。
- 第三方供应商或内部接口处理细节。
