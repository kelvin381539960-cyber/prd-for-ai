# 钱包 Playbook

## 适用场景

- 用户资产余额异常
- 充值不到账
- 发送失败
- 兑换失败
- 用户选择了错误网络、地址或币种

## 客服先确认

1. 用户问题类型：资产显示、充值、发送、兑换。
2. 币种、网络、金额、地址是否正确。
3. 是否有 TxID / 交易哈希。
4. 链上交易是否已提交或确认。
5. App 内交易状态是什么。

## 可向用户询问的信息

- 充值或交易币种
- 网络 / 链
- 金额
- 交易时间
- TxID / 交易哈希
- App 页面截图
- App 内交易状态

## 建议处理步骤

1. 先确认用户是否选对币种、网络和地址。
2. 如为充值，确认链上状态。
3. 引导用户刷新 App 并查看交易记录。
4. 如果链上已确认但 App 未到账，升级人工。
5. 如果疑似错链、错地址、错币种，不承诺找回，升级人工。

## 可使用的用户回复口径

```text
请先确认充值的币种、网络和地址是否正确，并查看链上交易是否已完成。如果链上已确认但 App 内长时间未到账，建议联系人工客服，并准备好交易哈希、币种、网络、金额和时间。
```

## 必须升级的情况

- 充值长时间不到账。
- 链上成功但 App 未到账。
- 错链、错地址、错币种。
- 资产余额异常。
- 用户要求追回、撤销或赔偿。

## 不要向用户透露

- 内部账务或风控逻辑。
- 未确认的支持币种、网络、手续费、限额。
- 找回或赔付承诺。
