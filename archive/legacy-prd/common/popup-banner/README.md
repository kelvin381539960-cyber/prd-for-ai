<p>运营弹窗</p>
<p>针对更新提示弹窗（强更or 弱更):</p>
<p>尺寸待UI稿输出后确认。（以UI为准）</p>
<p>由服务端配置实现</p>
<p>目标用户：全部；不区分人群</p>
<p>频控：</p>
<p>强更弹窗，无频控。</p>
<p>弱更弹窗，每天出现1次。用户手动关闭后，当天该弹窗不再出现。</p>
<p>针对运营弹窗：</p>
<p>尺寸待UI搞输出后确认。（343*456px@3x），需提供给Oboss-popup</p>
<p>由Oboss-popup提供能力实现。</p>
<p>支持目标用户配置，由Oboss提供能力实现。</p>
<p>具体人群配置，见：usergroup部分。</p>
<p>频控：</p>
<p>根据Oboss配置，下拉选项：</p>
<p>ONCE ONLY：一个user在popup有效期内仅第一次打开APP时弹出；</p>
<p>ONCE PER DAY：一个user在popup有效期内每天第一次打开APP时弹出。</p>
<p>ONCE THREE DAY：一个user在popup有效期内，每三天为一个周期，每个周期内第一次打开APP时弹出</p>
<p>ONCE PER WEEK：一个user在popup有效期内，每一周为一个周期，每个周期内第一次打开APP时弹出</p>
<p>ONCE PER MONTH：一个user在popup有效期内，每一个月为一个周期，每个周期内第一次打开APP时弹出</p>
<p>频率控制：读取OBoss pop up 配置中的 frequency 字段</p>
<p>前端和app需要新增频次处理逻辑：根据本地缓存上次弹窗展示过的时间，确定下一次展示时间，在到达下一次展示时间时，重新读取接口popup数据做展示。</p>
<p>运营弹窗内容</p>
<p>背景图从OBoss pop up 接口获取</p>
<p>按钮：从OBoss pop up 接口获取</p>
<p>跳转链接：从oboss pop up 接口获取</p>
<p>关闭按钮：前端写死固定在右上角；</p>
<p>若同时存在更新提示弹窗和运营弹窗：</p>
<p><strong>优先级：强更弹窗&amp;版本更新弹窗 &gt; 运营弹窗&gt;push引导开关弹窗。</strong></p>
<p><strong>频控：在首页每次展示，只会出现一个弹窗。（不会出现关闭弹窗再显示其他弹窗）</strong></p>
<p><strong>更新弹窗优先级最高，</strong>若满足显示条件，则<strong>不显示其他任何弹窗</strong>。显示条件：后台配置了更新弹窗（强更弹窗，或仅弹出 1 次的弱更弹窗），且首页接口数据更新时下发。</p>
<p><strong>运营弹窗优先级次之，</strong>若满足显示条件，则<strong>不显示 Push 引导弹窗</strong>。显示条件：当前无更新弹窗，或弱更弹窗已达频控上限；且运营弹窗未达频控上限，首页接口数据更新时下发。</p>
<p><strong>Push 引导弹窗（兜底）</strong>若以上两类弹窗均不显示，则显示 Push 引导开启弹窗。显示条件：</p>
<p>当前无更新弹窗，或弱更弹窗已达频控上限；</p>
<p>无运营弹窗，或运营弹窗已达频控上限；</p>
<p>设备 Push 开关未开启；</p>
<p><del>换句话表述：</del></p>
<p><del>更新弹窗出现，则不显示其他弹窗。（出现的条件是：配置了更新弹窗，强更弹窗或者仅显示1次弹出的弱更弹窗，首页接口数据更新时下发）</del></p>
<p><del>运营弹窗若出现，则不显示push引导弹窗。（出现的条件：当前无更新弹窗或者弱更新弹窗达到频控，且运营弹窗未达到频控限制，首页接口数据更新时下发）</del></p>
<p><del>如果上述都没有，则显示push 引导开启的弹窗（出现的条件：当前无更新弹窗或者弱更新弹窗达到频控，没有运营弹窗或者运营弹窗达到频控限制，push开关未开启，首页接口数据更新时下发）</del></p>
<p><del>免打扰控制：暂无。</del></p>
