# Biometric 生物识别

## 1. 功能概述

通过设备端生物识别（人脸/指纹）+ 后端签名验证的双重机制实现快捷登录和敏感操作确认。

## 2. 前置条件

| 条件 | 说明 |
|------|------|
| 设备已绑定 | 当前 Device ID 已与账户绑定 |
| 系统已启用 | 手机系统已开启人脸或指纹识别 |
| 用户已启用 | 用户在APP内完成BIO启用设置 |

## 3. 验证流程

```
用户触发BIO → 设备端验证 → 通过? 
  ├─ 是 → 后端验证BIO签名 → 通过?
  │       ├─ 是 → 验证成功，进入下一步
  │       └─ 否 → 弹窗错误 + 清除BIO + 跳转手动登录
  └─ 否 → 平台差异化处理（见下方）
```

## 4. 平台差异化处理

### 4.1 iOS 人脸识别 (Face ID)

| 失败次数 | 弹窗处理 |
|----------|----------|
| 单数次（1,3,5...） | 系统弹窗：`Try FaceID Again` / `Cancel` |
| 双数次（2,4,6...） | 弹窗：`Cancel` / `Use Other Methods` |

- `Try FaceID Again` → 重新拉起BIO验证
- `Cancel` → 关闭弹窗，回到登录页
- `Use Other Methods` → 跳转邮箱/手机登录页

### 4.2 iOS 指纹识别 (Touch ID)

| 失败次数 | 弹窗处理 |
|----------|----------|
| 每轮第1次 | 系统弹窗：`Cancel` |
| 每轮第2次 | 弹窗：`Cancel` / `Use another method` |
| 第5次（第3轮第1次） | 指纹被锁，引导用户使用其他方式 |

### 4.3 Android（人脸 & 指纹）

| 规则项 | 说明 |
|--------|------|
| 失败弹窗 | 系统弹窗，以各机型实际展示为准 |
| 失败次数限制 | 不限制，以各机型实际限制为准 |
| 超过设备限制 | 弹窗错误 + `Use another method` → 跳转手动登录 |
| 额外处理 | 清除本地BIO信息 + 隐藏 Quick Login 按钮 |

## 5. 后端验证失败处理

无论平台，后端验证失败统一处理：
1. 弹窗提示错误信息
2. 引导跳转至手动登录页面
3. 前端清除本地BIO信息
4. 后端关闭该账户BIO开关
5. 引导用户重新设置BIO

## 6. BIO 启用/关闭规则

| 场景 | BIO 状态变化 |
|------|-------------|
| 登录后引导启用 | 关→开 |
| 重置密码成功 | 开→关（自动关闭） |
| 后端验证失败 | 开→关（强制关闭） |
| 超过设备失败限制(Android) | 开→关（前端清除） |

## 7. 5分钟免验规则

- 用户完成手动登录后 **5分钟内**：设置BIO无需再次身份验证
- 超过5分钟：需先通过身份验证再设置BIO

## 8. 调用方

- `account/login` — Quick Login 快捷登录
- `account/login` — Enable BIO Page 引导启用
- 后续：卡交易确认、钱包转账确认等敏感操作
